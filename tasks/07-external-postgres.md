# M7：外部 PostgreSQL 模式与迁移

## 1. 目标

允许高级用户安全连接外部 PostgreSQL，并支持从托管本地数据库迁移过去。外部模式不得破坏托管数据库；迁移完成并验证前不得切换事实源。

## 2. 连接配置

保存连接配置：

- 主机；
- 端口；
- 数据库名；
- 用户名；
- TLS 模式；
- CA、客户端证书引用；
- 连接与查询超时；
- profile 名称。

密码和私钥不得明文写入普通配置文件或日志，使用系统安全凭据存储。

## 3. TLS

至少支持：

- 禁用：仅限明确警告的受信任本机/内网测试；
- 要求加密；
- 验证 CA；
- 验证 CA 和主机名。

默认推荐验证 CA 和主机名。证书错误不得静默降级为不验证。

## 4. 预检

连接外部数据库前验证：

- PostgreSQL 主版本受支持；
- `vector` 扩展已安装，或当前账号有权限创建；
- 当前账号可创建和修改所需 schema；
- 数据库可读写；
- UTF-8 编码；
- 时区和时间函数正常；
- 连接没有落在只读副本；
- 必需扩展、函数和索引能力可用；
- migration 表状态合法；
- 不存在不兼容的 ImageDB schema。

返回结构化诊断，不只返回一条连接失败字符串。

## 5. 外部数据库初始化

支持两种模式：

### 空数据库

- 创建 extension；
- 执行 migration；
- 写入实例标识；
- 完整健康检查。

### 已有 ImageDB 数据库

- 读取 schema version；
- 执行可升级 migration；
- 禁止自动降级；
- 未知或更高版本拒绝连接；
- 验证数据库身份。

## 6. 托管到外部迁移

流程：

1. 外部数据库预检；
2. 阻止新的写任务；
3. 等待当前文件事务到达安全状态；
4. 创建托管数据库备份；
5. 创建外部目标 schema；
6. 导出托管数据库；
7. 导入外部数据库；
8. 执行 migration；
9. 验证表数量、行数、约束、索引和关键 Hash；
10. 验证图库根、图集、图片、计划、事务和设置记录；
11. 在外部数据库执行读写冒烟；
12. 原子切换 active database profile；
13. 保留托管数据库作为回滚点；
14. 用户确认后才允许清理旧备份。

## 7. 回滚

以下任一情况必须保持托管数据库为 active：

- 网络中断；
- TLS失败；
- 导入失败；
- migration失败；
- 验证不一致；
- 外部数据库转为只读；
- 用户取消。

切换后首次启动外部数据库失败时，提供受控回退到托管数据库，不自动覆盖外部数据。

## 8. 运行时行为

- 连接池健康检查；
- 指数退避重连；
- 明确区分连接失败、认证失败、TLS失败、权限失败和只读；
- 网络失败期间禁止虚假写成功；
- 当前数据库不可用时，文件提交不得继续到数据库确认阶段；
- 已发布目录通过现有 Recovery 恢复数据库提交。

## 9. GUI

提供：

- 新建和编辑连接 profile；
- 测试连接；
- 展示 TLS 和证书结果；
- 权限和扩展预检；
- 托管到外部迁移向导；
- 迁移进度；
- 验证报告；
- 失败回滚；
- 当前 active profile；
- 受控切回托管数据库。

## 10. 测试

覆盖：

- 正常 TLS；
- CA错误；
- 主机名错误；
- 密码错误；
- 缺少 vector；
- 无创建 extension 权限；
- 无 schema 权限；
- 只读数据库；
- 迁移中断；
- 导入后行数不一致；
- 切换前网络断开；
- 切换后首次启动失败；
- 回退到托管数据库；
- 外部数据库恢复后继续文件事务。

## 11. 完成定义

- 外部连接默认支持严格 TLS；
- 凭据安全保存；
- 预检完整；
- 迁移前备份；
- 验证通过后才切换；
- 失败可回退；
- 托管数据库不被破坏；
- GUI、测试和文档完整。
