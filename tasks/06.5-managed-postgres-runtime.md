# M6.5：托管 PostgreSQL 运行时与 Windows 安装包

## 0. 当前状态

截至 2026-07-04，仓库已记录托管 PostgreSQL 在本地开发运行时下通过真实 PostgreSQL 18.4 + pgvector 0.8.3 验证，并已进入 `tasks/07-external-postgres.md`。

本文件不再作为当前执行任务，而作为发布复验和回归排查依据保留。若 M7/M8/M9 中发现托管运行时、安装包资源或无预装数据库场景回归，应回到本文件对应条目补齐。

## 1. 目标

默认本地模式随 ImageDB 安装包提供 PostgreSQL 与 pgvector 运行时。最终用户只需安装并启动 ImageDB，不需要系统级 PostgreSQL、Docker 或数据库管理工具。

## 2. 非目标

- 不把数据库数据放进应用安装目录。
- 不依赖系统 PATH 中的 PostgreSQL 作为正式默认路径。
- 不在本阶段实现外部数据库迁移。
- 不在本阶段实现共享目录兼容。

## 3. 运行时目录布局

建议资源布局：

```text
resources/
└─ postgres-runtime/
   ├─ runtime-manifest.json
   ├─ bin/
   ├─ lib/
   └─ share/
      └─ extension/
```

`runtime-manifest.json` 至少包含：

- PostgreSQL 主版本和完整版本；
- pgvector 版本；
- 构建平台与架构；
- 必需文件列表；
- 每个文件大小和 BLAKE3；
- 运行时包格式版本。

不得依赖“目录存在”判断运行时完整。

## 4. 构建与供应链

建立独立、可重复执行的运行时准备脚本：

1. 获取项目固定版本的 PostgreSQL Windows 运行文件；
2. 获取与该 PostgreSQL 主版本匹配的 pgvector 扩展；
3. 仅保留托管数据库所需文件；
4. 生成完整性清单；
5. 生成第三方许可证与 NOTICE；
6. 将结果复制到 Tauri resources；
7. 构建期间验证缺失文件和 Hash；
8. 任何文件不一致时构建失败。

运行时来源、版本和 Hash 必须固定在仓库配置中，不能在用户首次启动时从网络下载。

## 5. 运行时定位

正式优先级：

```text
Tauri resource_dir/postgres-runtime
```

开发模式可支持：

```text
IMAGEDB_POSTGRES_BIN
仓库内开发运行时目录
系统安装目录或 PATH（仅开发兜底）
```

正式安装包不得因为系统恰好安装 PostgreSQL 而偷偷使用系统版本。

运行时解析结果必须记录：

- 来源；
- 版本；
- 路径；
- 完整性验证结果；
- pgvector 文件验证结果。

## 6. 应用数据布局

建议：

```text
<AppData>/ImageDB/
├─ managed-postgres/
│  ├─ data/
│  ├─ logs/
│  ├─ backups/
│  ├─ runtime-state.json
│  └─ instance.lock
└─ app/
```

要求：

- 安装目录只读也能运行；
- 卸载应用不会默认删除数据库数据；
- 数据目录支持非 ASCII 用户名；
- 所有路径使用绝对路径并进行规范化；
- 不把密码写入日志。

## 7. 首次初始化

流程：

1. 获取单实例锁；
2. 验证运行时；
3. 选择安全本地端口；
4. 生成随机数据库账号和密码；
5. 创建临时初始化目录；
6. 执行 `initdb`；
7. 写入仅监听 loopback 的配置；
8. 启动 PostgreSQL；
9. 创建应用数据库；
10. 创建 `vector` 扩展；
11. 执行全部 migration；
12. 执行读写和向量健康检查；
13. 原子写入实例状态；
14. 初始化成功后才把模式切换为托管本地。

任一步失败：

- 不显示成功；
- 保留可诊断日志；
- 清理仅属于本次初始化的临时目录；
- 不破坏既有数据库；
- 允许重新初始化。

## 8. PostgreSQL 配置

至少要求：

- 仅监听 `127.0.0.1` 与 `::1`；
- 禁止远程访问；
- 使用随机本地端口；
- 明确 UTF-8 编码；
- 日志轮转；
- 数据目录和日志目录独立；
- 合理的桌面级内存配置；
- 禁止使用默认弱口令；
- 配置文件由应用管理并带 schema version。

## 9. 生命周期管理

实现：

- 应用启动时检查数据库状态；
- 进程不存在时安全启动；
- PID 文件过期时清理；
- 端口被占用时重新分配或报告冲突；
- 应用退出时请求优雅停止；
- 异常退出后下次启动执行恢复检查；
- 数据库启动超时和停止超时；
- 禁止并发初始化；
- 多个 ImageDB 实例共享同一数据目录时，只有一个实例拥有数据库生命周期控制权。

不允许通过进程名称批量结束系统 PostgreSQL。

## 10. 健康检查

健康检查必须验证：

- 进程存在；
- 端口可连接；
- 当前连接的是预期实例；
- PostgreSQL 主版本兼容；
- `vector` 扩展存在且版本兼容；
- schema migration 版本正确；
- 数据库可读写；
- 简单向量操作可执行；
- 数据目录与实例 ID 一致。

## 11. 备份和恢复

实现托管数据库备份：

- 用户手动备份；
- 大版本迁移前强制备份；
- 输出文件带 manifest、数据库版本、schema 版本和 Hash；
- 备份写入临时文件后原子发布；
- 恢复前验证文件和兼容性；
- 恢复失败时保留原数据库。

## 12. 运行时升级

区分：

### 同一主版本更新

- 停止数据库；
- 备份；
- 切换运行时；
- 启动并检查；
- 失败时回退旧运行时。

### PostgreSQL 大版本升级

- 不直接用新二进制打开旧数据目录；
- 使用逻辑导出与恢复或受控升级流程；
- 先建立新数据目录；
- 完整验证后切换；
- 保留旧数据目录直到用户确认。

## 13. Windows 安装包

启用 Tauri bundle，安装包必须包含：

- ImageDB 程序；
- PostgreSQL 运行时；
- pgvector；
- 许可证和 NOTICE；
- 运行时 manifest。

验收：

- 无管理员权限的普通用户可安装到用户目录；
- 安装路径含空格和非 ASCII 字符；
- Windows Defender 开启；
- 系统未安装 PostgreSQL；
- 系统已安装其他 PostgreSQL 时不发生串用；
- 卸载后用户数据库默认保留并明确提示。

## 14. GUI

数据库设置页显示：

- 模式：托管本地；
- 运行时版本；
- 数据库版本；
- pgvector 版本；
- 数据目录；
- 端口；
- 运行状态；
- 最近备份；
- 诊断与日志入口。

按钮：

- 初始化；
- 启动；
- 停止；
- 重启；
- 健康检查；
- 创建备份；
- 恢复备份。

运行时缺失时显示：

```text
当前安装包缺少托管数据库运行文件，请重新安装完整版本。
```

不得要求普通用户“安装 PostgreSQL”。

## 15. 测试矩阵

### 单元测试

- 运行时清单解析；
- Hash 验证；
- 路径解析；
- 端口选择；
- 状态机；
- 配置生成；
- 版本兼容判断。

### 集成测试

- 空数据目录初始化；
- 初始化中断；
- extension 创建失败；
- migration 失败；
- 端口冲突；
- stale PID；
- 数据库进程崩溃；
- 备份与恢复；
- 同主版本运行时升级；
- 非 ASCII 路径。

### 干净环境验收

必须在未安装 PostgreSQL 的干净 Windows 虚拟机执行：

```text
安装
→ 启动
→ 初始化托管数据库
→ 创建图库
→ 扫描与入库小样本
→ 退出
→ 重启
→ 数据仍存在
→ 备份
→ 卸载并重新安装
→ 恢复并验证
```

## 16. 完成定义

- 安装包自带完整、可验证的 PostgreSQL 与 pgvector 运行时；
- 干净 Windows 无需安装数据库；
- 运行时和数据目录分离；
- 仅监听本机；
- 初始化、启停、崩溃恢复、备份和恢复可用；
- 运行时缺失时给出安装包错误；
- 全部测试与安装包验收通过。
