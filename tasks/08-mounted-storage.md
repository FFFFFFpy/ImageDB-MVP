# M8：已挂载共享存储兼容

## 1. 目标

支持操作系统已经挂载的共享路径作为图库目录。应用把它视为文件系统路径，不内置 SMB 协议客户端。

## 2. 非目标

- 不管理 NAS 用户和密码；
- 不负责挂载盘符或网络卷；
- 不实现 SMB 协议；
- 不假设共享存储具备本地 NTFS 的全部语义。

## 3. 存储能力探测

新增 `StorageCapabilities`，至少探测：

- 可读；
- 可写；
- 可创建目录；
- 同目录文件 rename；
- 同根目录 rename；
- 目录 rename 是否可用；
- overwrite rename 行为；
- 文件 `sync_all`；
- 父目录持久化同步；
- 大小写敏感性；
- Unicode 正规化行为；
- 最大路径和文件名；
- 文件锁能力；
- 时间戳精度；
- 可用空间；
- 设备或卷身份。

探测必须使用专用临时目录并完整清理。无法确定的能力应标记为 unknown，不能乐观假设支持。

## 4. 发布策略

根据能力选择：

### StrongLocal

条件：同根目录原子目录 rename 和必要持久化语义可用。

流程继续使用：

```text
完整 staging
→ 目录 rename
→ 正式目录一次出现
```

### ConservativeMounted

适用于无法证明目录 rename 原子性的挂载存储。

流程：

1. 在图库根内创建 transaction staging；
2. 完整复制和校验；
3. 写 manifest；
4. 将文件发布到目标目录；
5. 最后写入不可变提交标记；
6. 只有存在合法提交标记的目录才被 ImageDB 视为已发布；
7. 中断目录进入 Recovery，不得显示完成。

提交标记至少绑定：

- transaction ID；
- plan hash；
- manifest hash；
- 完整文件集合；
- 发布策略版本。

该策略不应被描述为原子发布，只能描述为可检测、可恢复发布。

### Unsupported

若无法保证基本写入、验证和冲突保护，拒绝将该路径设为图库根。

## 5. 多实例保护

使用数据库级图库根租约作为权威锁：

- `library_root_id`；
- owner instance ID；
- lease token；
- heartbeat；
- expires_at。

存储上的 lock 文件只作为诊断辅助，不作为唯一锁，因为共享文件系统的锁语义可能不可靠。

要求：

- 同一图库根只有一个写入者；
- 其他实例可以只读或被明确拒绝；
- lease 过期后需要验证旧进程和未完成事务；
- 不得简单删除未知 lock 文件后继续写入。

## 6. 路径规则

所有路径必须：

- 相对于持久化图库根；
- 禁止 `..`；
- 禁止绝对 target path；
- 规范化分隔符；
- 检测大小写折叠冲突；
- 检测 Unicode 正规化冲突；
- 检测 Windows 保留名称；
- 检测尾随点和空格；
- 检测路径长度；
- 不跟随符号链接或 reparse point 逃逸图库根。

## 7. 断连与恢复

处理：

- 挂载点消失；
- 网络断开；
- 文件读写超时；
- 共享目录变为只读；
- 空间不足；
- 设备重新挂载到不同路径；
- 同一路径指向不同设备；
- staging存在但目标不存在；
- 目标存在但提交标记缺失；
- 文件集合不完整；
- reconnect后继续验证和恢复。

恢复前必须重新探测设备身份和存储能力。不能只因为路径字符串相同就继续。

## 8. 性能与限流

- 文件复制并发可配置；
- 默认共享存储低并发；
- Hash和复制流式执行；
- 限制同时打开文件数量；
- 支持暂停和安全取消；
- 进度显示实际字节数和重试；
- 网络超时不阻塞 GUI；
- 不对共享目录执行无界并行扫描。

## 9. GUI

选择图库路径后显示能力报告：

```text
存储类型：本地 / 挂载共享 / 未知
发布策略：强一致 / 保守可恢复 / 不支持
目录重命名：支持 / 不支持 / 未知
持久化同步：支持 / 降级 / 不支持
大小写：敏感 / 不敏感
可用空间：...
```

明确提示：

- 共享存储断开会暂停任务；
- 不要让多个 ImageDB 实例同时写同一图库；
- 保守策略下，没有提交标记的目录不视为正式图库内容。

## 10. 测试矩阵

### 能力探测

- 本地 NTFS；
- 映射盘符；
- UNC 挂载后路径；
- 大小写不敏感；
- Unicode文件名；
- 长路径；
- 目录 rename 不可靠模拟；
- sync不支持模拟；
- 锁不可靠模拟。

### 故障注入

- 复制中断网；
- manifest后断网；
- 发布一半断网；
- 提交标记前断网；
- 提交标记后数据库失败；
- 空间不足；
- 文件被外部修改；
- 目标目录由其他用户创建；
- lease丢失；
- 重新挂载到不同设备。

### 真实环境门禁

至少使用一个真实共享存储挂载执行：

```text
扫描
→ 审核
→ 冻结计划
→ 提交
→ 人为断网
→ 重新挂载
→ Recovery
→ 完整Hash验证
→ 数据库和文件一致
```

## 11. 完成定义

- 存储能力有结构化探测结果；
- 不对未知能力做乐观假设；
- 发布策略明确分级；
- 断连后不产生虚假完成；
- 多实例写入受数据库租约保护；
- 路径冲突和逃逸受控；
- 真实挂载环境故障测试通过。
