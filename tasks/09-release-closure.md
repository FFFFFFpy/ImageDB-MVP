# M9：端到端发布收口

> 收口状态（2026-07-05）：M6.5–M9 主链接通并经真实 PostgreSQL + 真实文件系统验证。冻结计划主链（Review → `freeze_import_plan` → Commit 读取 frozen summary）一致；可提交 run 查询优先 `ready_to_commit`；真实测试缺 runtime 时 fail-fast。最终验证命令与结果见 `reports/m6_5_m9_closure.md`。

## 1. 目标

把已完成的数据库、扫描、审核、入库、恢复和共享存储能力收口为可安装、可升级、可诊断的 Windows MVP。

## 2. 公开主链验收

必须通过真实 GUI/IPC 路径：

```text
首次启动
→ 初始化托管数据库
→ 选择源目录
→ 选择图库根
→ 扫描
→ 查重
→ 人工审核或零候选
→ 冻结计划
→ 提交确认
→ staging
→ 校验
→ 发布
→ 数据库确认
→ 源归档
→ 完成
```

不得通过测试辅助函数手工插入计划或事务来替代生产主链。

## 3. 数据集

准备固定验收数据：

- 完全重复文件；
- 同像素不同编码；
- 感知相似图片；
- 跨图集重复；
- 与历史图库重复；
- 被排除图片；
- 嵌套目录；
- 侧车文件；
- Unicode与长路径；
- 损坏图片；
- 超大图片；
- 空图集；
- 大图集。

所有数据附带预期结果清单。

## 4. 安装与升级

验证：

- 干净 Windows 安装；
- 无管理员权限安装；
- 非 ASCII 用户目录；
- 系统无 PostgreSQL；
- 系统有其他 PostgreSQL；
- 覆盖安装升级；
- 应用卸载后保留数据；
- 重装后识别旧托管数据库；
- 运行时小版本升级；
- 数据库备份与恢复。

## 5. 故障矩阵

至少覆盖：

- 扫描中取消；
- 复制中取消；
- 复制后崩溃；
- manifest写入后崩溃；
- 发布后数据库失败；
- 数据库成功后源归档失败；
- 应用强制终止；
- PostgreSQL强制终止；
- 磁盘空间不足；
- 图库目录被外部修改；
- 网络存储断连；
- 外部数据库断连；
- 机器重启后恢复。

## 6. 性能

记录并建立最低门禁：

- 1千、1万、10万图片扫描；
- Hash吞吐；
- 内存峰值；
- 历史图库召回查询；
- 候选生成；
- GUI响应；
- 大文件复制；
- Recovery扫描耗时；
- PostgreSQL启动时间。

禁止无界内存加载和全库笛卡尔比较。

## 7. 稳定性

执行：

- 长时间空闲；
- 多轮连续导入；
- 多次启动退出；
- 24小时稳定性运行；
- 多次备份与恢复；
- 重复执行Recovery；
- 日志轮转。

## 10. 发布门禁

最终必须运行：

```powershell
pnpm install
pnpm format:check
pnpm typecheck
pnpm test:unit
pnpm rust:test
pnpm rust:clippy
pnpm rust:test:real
pnpm build
pnpm release:gate
```

`pnpm rust:test:real` 缺 PostgreSQL runtime 时必须失败（不得 skip）；
`pnpm release:gate` 在构建后运行 `release:verify-artifacts` 校验打包的
runtime 完整性。

另需：

- 干净 Windows 安装测试；
- 托管数据库无预装环境测试；
- 外部 PostgreSQL TLS测试；
- 真实共享存储故障测试；
- 完整 GUI闭环；
- 安装包签名与版本信息检查。
