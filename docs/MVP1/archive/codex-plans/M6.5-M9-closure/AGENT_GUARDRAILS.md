# Agent 执行边界

## 不要做的事

- 不要重写整个数据库层。
- 不要重写整个 UI。
- 不要重写扫描、指纹、重复检测、事务恢复主链。
- 不要引入新数据库、新语言、新后台服务。
- 不要实现 SMB 协议，只支持已经挂载为本地路径的目录。
- 不要追求 PostgreSQL 运行时跨平台全覆盖，本轮只保证 Windows x64。
- 不要做长期 CI/CD 平台化。
- 不要写大量空报告和流程文档替代真实接线。
- 不要因为“还有更完美的安全模型”而无限扩展。
- 不要扣命名、注释、格式这种不阻断主链的细节。
- 不要修无关 lint、无关样式、无关重构。
- 不要在当前轮次继续 M7/M8/M9 的新功能扩张。

## 必须做的事

- 解决默认本地模式缺 PostgreSQL runtime 的问题。
- 让 release 构建产物包含 PostgreSQL + pgvector runtime。
- 让真实测试缺 runtime 时 fail-fast，而不是 skip。
- 让外部 PostgreSQL 服务主链使用已有 TLS connector。
- 让提交页读取 frozen plan，而不是动态重算。
- 修复可提交 run 查询优先级。
- 补最小必要真实测试。
- 更新任务与报告，让项目状态可信。

## 判断是否完成

完成不是“写了脚本”。完成是：

```text
干净 Windows 环境
→ 不安装 PostgreSQL
→ 启动 ImageDB
→ 初始化托管数据库
→ 扫描
→ 审核/零候选
→ 冻结计划
→ 提交
→ 恢复可用
→ completed
```

如果这条线不通，别写“完成”。
