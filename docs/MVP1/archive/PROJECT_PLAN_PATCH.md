## M6.5 托管 PostgreSQL 运行时与安装包

目标：默认本地模式随应用提供 PostgreSQL 与 pgvector 运行时，用户无需预装数据库。

交付内容：

- 受版本清单约束的 PostgreSQL 运行时资源；
- pgvector 扩展运行文件；
- Tauri 资源定位与完整性验证；
- 首次启动自动初始化；
- 生命周期、崩溃恢复、端口冲突和单实例保护；
- 数据备份、运行时升级和数据库大版本迁移策略；
- Windows 安装包；
- 干净 Windows 环境验收。

完成标准：未安装 PostgreSQL 的普通 Windows 用户安装应用后，可直接初始化并使用托管数据库。

计划状态：作为发布复验基线保留；当前继续执行 M7，M9 必须重新覆盖 M6.5 的安装包级验收。

## M7 外部 PostgreSQL

目标：允许高级用户连接受信任的外部 PostgreSQL，并安全迁移托管数据库。

交付内容：

- 外部连接配置与安全凭据存储；
- TLS、证书和主机名校验；
- 版本、pgvector、权限、编码和可写性预检；
- 外部数据库迁移与健康检查；
- 托管数据库到外部数据库的备份、导出、恢复、验证和切换；
- 失败回滚与重新连接；
- GUI 配置、迁移向导和诊断。

完成标准：迁移成功前不切换连接；迁移失败时托管数据库仍可继续使用。

## M8 已挂载共享存储

目标：支持 Windows/macOS/Linux 已经挂载的共享目录作为图库路径，不内置 SMB 客户端。

交付内容：

- 存储能力探测；
- 原子目录重命名、持久化同步、锁、大小写和路径能力分级；
- 本地强一致发布策略；
- 共享存储保守发布策略与最终提交标记；
- 断连、超时、空间不足和重新挂载恢复；
- 路径规范化、Unicode、长路径和大小写冲突处理；
- 多实例写入保护；
- 真实挂载环境验收。

完成标准：断连或重试不会覆盖未知文件、产生虚假完成状态或丢失源文件。
