# M3 产品与体验简报

## 1. 任务定义

M3 重设计 ImageDB 桌面端全部现有页面，服务于需要整理本地图片图集、长时间观察分析进度、人工判断重复图片并安全入库的用户。

用户进入产品时可能处于三种状态：准备开始一次整理任务、等待长任务完成、处理需要谨慎决策的异常。界面需要让用户始终知道“系统现在在做什么、是否安全、我下一步做什么”。

成功标准不是“看起来更漂亮”，而是：

- 新用户能从工作台无解释地开始第一轮导入。
- 返回用户能一眼找到当前任务与后端给出的下一步动作。
- 分析、审核、计划、入库和恢复之间的状态不会被视觉层混淆。
- 图片是主要内容，技术字段按需出现，不与主要决策竞争注意力。
- 长任务期间界面稳定、有进度、有取消反馈，不制造“是否卡死”的焦虑。

## 2. 视觉方向

### 色彩策略

采用 **Restrained（克制）** 的产品色彩策略：中性浅色工作面承载大部分内容，森林绿只用于主操作、当前选择与成功状态；蓝色用于信息和技术链接，橙色用于等待与警告，红色只用于错误或破坏性操作。

不采用满屏高饱和游戏配色，也不把所有内容装进彩色卡片。

### 场景句

用户在室内普通显示器前连续整理数百到数万张图片，需要一个白天与夜间环境都不刺眼、长时间观看仍安静、在关键文件操作上值得信任的浅色桌面工具。

### 视觉锚点

- 用户提供的 ImageDB 重设计母版：布局、任务节奏、图片优先、明确下一步。
- `animal-island-ui`：圆润轮廓、自然色彩、轻微按压反馈、友好但不幼稚。
- Linear / Notion 类产品工具：可预测的导航、克制的信息密度、标准化交互状态。

## 3. 体验原则

### 图片优先

缩略图、对比图、图集封面优先于 UUID、hash 和路径。技术信息仍然可达，但默认位于次级区域、详情面板或可展开诊断中。

### 单任务主线

每个页面只突出一个主动作。主要工作流为：

```text
工作台
→ 新建导入 / 继续分析
→ 图集进度
→ 审核
→ 生成并检查导入计划
→ 确认入库
→ 完成或恢复
```

页面不重复提供多个同等级绿色 CTA。后退、取消、跳过和危险操作保持明确的视觉层级。

### 状态可信

- 状态文案来自持久化事实或后端显式返回值。
- 颜色不能是唯一状态信号；必须同时有图标、文字或进度值。
- “处理中”“等待审核”“可恢复”“终态失败”不能合并为一个模糊的“异常”。
- 成功页只在真实操作完成后出现，不用乐观 UI 伪装文件事务结果。

### 轻量反馈

主按钮按下时允许 1–2px 的纵向位移与底部深色层收缩；列表选择、状态变化和面板切换使用 150–220ms 的 ease-out。禁止与状态无关的页面入场编排、弹跳或持续装饰动画。

### 渐进披露

首页展示用户能理解的图库与任务摘要。migration version、端口、数据库目录、hash、transaction operation 等技术信息进入“系统状态”“详情”或“诊断”区域。

## 4. 信息架构

### 主导航

固定为六个用户任务入口：

| 导航       | 目的                                   | 备注                                       |
| ---------- | -------------------------------------- | ------------------------------------------ |
| 工作台     | 当前状态、最近任务、后端指定的下一步   | 默认入口                                   |
| 新建导入   | 选择源目录、发现图集、开始/继续分析    | 同一页面承载 run 上下文                    |
| 审核       | 处理需要人工判断的候选                 | 有待办时显示计数徽标                       |
| 导入计划   | 查看、调整并冻结计划                   | 现有 `commit` 路由在 M3 中使用用户语言命名 |
| 入库与恢复 | 执行入库、查看进度、处理恢复或失败事务 | 可在页面内按事实分区，不改变后端语义       |
| 设置       | 数据库、图库位置、诊断和高级选项       | 技术探针移入高级设置                       |

“技术探针”不继续占用一级导航。它属于设置中的高级诊断，现有功能保留。

### 页面层级

每个页面统一为：

```text
页面标题 + 一句话状态
→ 主任务区
→ 次级事实 / 详情
→ 固定或就近的主要动作
```

不在主画布顶部堆叠数据库、pgvector、migration 三张同权卡片。系统健康状态在工作台顶部使用一个紧凑状态条；详情进入展开面板。

## 5. 范围

### 包含

- 全局应用壳、导航、页面标题和内容宽度规则。
- 工作台、新建导入、分析进度、审核、导入计划、入库、恢复、设置、首次配置。
- 颜色、字体、间距、圆角、层级、图标和动效 token。
- 基础组件与状态组件的统一行为。
- 空、加载、部分数据、成功、警告、错误、取消、不可恢复状态。
- 键盘操作、焦点、缩放、窄窗口与高密度数据适配。
- 现有测试的选择器迁移和新增 UI 行为测试。

### 不包含

- 新匹配算法、新图库搜索、新标签系统或云能力。
- 数据库 schema 或 migration 改造。
- run / album / candidate / frozen plan / file transaction 状态语义改造。
- 图集级 Commit。
- 仅为新外观而修改 Rust domain 或 repository。
- 超出 CC BY-NC 4.0 允许范围使用、再许可或去除 attribution。

## 6. 内容与真实范围

界面至少覆盖以下规模：

| 数据       | 空状态 |       常见 |             压力范围 |
| ---------- | -----: | ---------: | -------------------: |
| 图集       |      0 |       5–50 |               1,000+ |
| 单次图片   |      0 | 100–10,000 |              100,000 |
| 待审核候选 |      0 |      1–100 |              10,000+ |
| 错误条目   |      0 |       1–20 |               1,000+ |
| 路径文本   | 短英文 |  中文/空格 | 长路径与不可断词片段 |

大列表不得一次把所有重内容渲染到 DOM。实施阶段应先测量，达到可见卡顿时采用分页或虚拟化，但不能通过截断数据改变完整召回与业务语义。

## 7. 关键风险

- 风格过度游戏化，降低文件事务和错误恢复的可信度。
- 只重做首页，后续高密度页面仍使用旧组件词汇，造成视觉断裂。
- 为了“简化”而隐藏 frozen plan、取消、失败或恢复所需的证据。
- 在 React 中根据计数重新推断 `next_action`。
- 使用组件库时遗漏 attribution、许可证文件或非商业用途说明。
- 只验证浏览器版本，没有在 Tauri 窗口尺寸、文件选择和真实图片预览中验证。
