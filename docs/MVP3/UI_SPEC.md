# M3 UI 规范

## 1. 总体布局

### 应用壳

- 左侧固定导航：默认 216px；紧凑窗口下收起为 64px 图标栏。
- 主内容区：背景使用接近中性的浅色，内容最大宽度按页面类型控制。
- 任务页最大宽度建议 1180–1440px；审核对比页可以占满可用宽度。
- 主要内容采用 8px 基础间距系统，页边距桌面 24–32px、窄窗口 16px。
- 顶部不增加全局大标题栏；窗口空间留给当前任务。

### 导航

- 品牌区显示 ImageDB 名称与简洁图片/图集图标，不使用受版权限制的游戏素材。
- 当前项使用柔和绿色底、深绿色文字和图标；不使用整块高饱和色。
- 待审核、失败或恢复数量使用小型徽标，徽标必须有可读文本或 accessible name。
- 设置固定在导航底部；用户/账户占位在没有真实账户功能前不出现。

## 2. 设计 token 基线

实现时应先落地语义 token，再改页面。以下值是首轮基线，可在真实截图对比后微调，但不得在各页面散写颜色。

### 颜色

| Token                     | 建议值    | 用途                 |
| ------------------------- | --------- | -------------------- |
| `--color-canvas`          | `#F5F7F4` | 应用背景             |
| `--color-surface`         | `#FFFFFF` | 主工作面             |
| `--color-surface-soft`    | `#EEF4EE` | 选中、辅助区         |
| `--color-ink`             | `#172019` | 主文字               |
| `--color-ink-muted`       | `#59635B` | 次级文字，需满足 AA  |
| `--color-border`          | `#DCE4DD` | 标准边框             |
| `--color-primary`         | `#127C42` | 主 CTA、当前状态     |
| `--color-primary-hover`   | `#0F6F3B` | 主 CTA hover         |
| `--color-primary-pressed` | `#0C6033` | 主 CTA active / 底层 |
| `--color-info`            | `#3267B1` | 信息、链接           |
| `--color-warning`         | `#B86B13` | 等待、需注意         |
| `--color-danger`          | `#B63D3D` | 错误、破坏性操作     |
| `--color-success-soft`    | `#EAF6EE` | 成功背景             |
| `--color-warning-soft`    | `#FFF4E3` | 警告背景             |
| `--color-danger-soft`     | `#FCECEC` | 错误背景             |

正文与背景对比度至少 4.5:1；大文字至少 3:1；focus ring、状态图标和边界也需可辨认。最终实现以自动对比度检查结果为准。

### 字体

- UI 字体：`Inter, "PingFang SC", "Microsoft YaHei", system-ui, sans-serif`。
- 不把外部字体下载作为应用启动前提；离线安装包必须完整可用。
- 页面标题 22px / 700；区块标题 16px / 700；正文 14px / 400–500；标签和表格 12–13px。
- 数据使用 tabular numbers；路径、hash、migration 使用系统等宽字体。
- 不使用展示字体、超大标题、全大写英文眉题或过密负字距。

### 形状与层级

- 控件圆角：10–12px；按钮允许 12px 或胶囊形，但同一层级保持一致。
- 容器圆角：16px；图片圆角：10–12px。
- 卡片默认不依赖大阴影，通过边框和背景分层；浮层使用轻阴影。
- 主要按钮可使用 3px 底部深色层表达按压，active 时收缩为 1px。
- 不使用玻璃拟态、渐变文字或装饰性大面积渐变。组件库中高度游戏化的不规则弹窗仅用于轻量、非危险提示；Commit、Recovery 和破坏性确认使用边界清晰的标准对话框。

### 动效

- hover / press：150ms；面板与状态转换：180–220ms。
- 使用 ease-out 曲线，只动画 transform、opacity 或必要的可合成属性。
- 进度条可以平滑更新，但不能用无限动画掩盖停滞。
- `prefers-reduced-motion: reduce` 下取消位移与循环动画，保留即时状态切换。

## 3. 基础组件

优先评估 `animal-island-ui` 已提供的 Button、Card、Input、Switch、Checkbox、Radio、Tooltip、Modal、Drawer、Table、Progress 等组件。所有第三方组件通过 `components/ui/` 内的 ImageDB 适配组件导出，以统一中文文案、尺寸、语义 token、可访问性和业务需要的状态。

以下内容不应为了“统一使用组件库”而强行套用通用组件：审核双图画布、图集进度行、frozen plan 图集树、Commit 阶段进度和 Recovery 事务证据面板。这些是 ImageDB 的领域界面，应独立实现并复用同一套 token。

### Button

- `primary`：每个页面主要操作，最多一个同等级主按钮。
- `secondary`：返回、查看详情、非主流程操作。
- `quiet`：工具栏、图片缩放、展开诊断。
- `danger`：放弃任务、不可逆清理；需明确对象并在必要时确认。
- 所有变体具备 default、hover、focus-visible、active、disabled、loading。
- loading 期间保持按钮宽度，显示动作文本，例如“正在生成计划…”，不只显示旋转图标。

### StatusBadge / StatusBanner

- Badge 用于行内短状态；Banner 用于需要行动的页面级事实。
- 状态由图标 + 文案 + 颜色共同表达。
- 错误 Banner 提供可执行下一步和诊断入口；不只显示原始异常字符串。

### Progress

- 同时显示任务名、已完成/总数、百分比和当前对象。
- 总数未知时使用明确的 indeterminate 状态，并写“正在统计”，不能显示虚假的 0%。
- 超过合理时间无更新时保持真实值，并提供取消或查看详情，不自动判定失败。

### Table / AlbumList

- 表头可粘性固定；长路径中间省略并通过 tooltip/复制获得完整值。
- 主要列在窄窗口保留，次要列折叠进详情。
- 行操作不全部做成彩色按钮；主操作一个，其余进入文字按钮或菜单。
- 图片列表优先使用缩略图 + 名称 + 核心状态，不用无意义的卡片网格。

### Empty / Loading / Error

- 加载使用与真实布局对应的 skeleton，不用页面中央孤立 spinner。
- 空状态说明为什么为空，并提供下一步；“待审核为 0”应表达任务已清空而非错误。
- 错误状态保留可复制的错误码/详情入口，同时给用户语言的处理建议。

## 4. 页面规范

### 工作台

布局顺序：

1. 欢迎语或当前工作状态。
2. 一个“下一步”主任务面板，CTA 直接映射后端 `next_action`。
3. 最近任务：图集缩略图、进度、待审核/失败摘要。
4. 图库概览：已入库图集、图片、图库位置，并提供“查看图库明细”二级入口。
5. 紧凑系统健康条；数据库模式、pgvector、migration 详情默认折叠。

禁止前端根据计数覆盖后端 CTA。没有 actionable run 时主动作才是“新建导入”。

### 图库明细

- 只展示 `committed` 图集与图片；不提供重命名、移动、删除或文件系统写入操作。
- 从工作台图库概览进入，返回时仍落在工作台；不改变主工作流导航与后端 `next_action`。
- 顶部汇总已入库图集数、图片数与登记大小；主体使用接近审核清单的信息密度，逐个图集展开图片明细。
- 图集按每批 50 个读取，展开后图片按每批 24 张读取；不能为展示明细一次挂载整个图库。
- 明确区分 loading、error、empty 与成功数据；错误时可以重试，空图库引导新建导入。
- 图集行显示名称、图库根目录、相对路径、已提交图片数、登记大小和提交时间；图片行显示相对路径、尺寸、格式、大小与已入库状态。

### 新建导入 / 分析

- 选择目录是页面首要动作；路径输入只读，点击整行或“选择目录”打开系统选择器。
- 发现图集后显示封面、名称、图片数和校验摘要。
- 分析前明确“此阶段不会修改源文件”。
- 分析中显示总进度、当前图集、图集状态列表、取消入口。
- failed 图集提供单图集 retry；abandoned 历史只读，不出现 resume/retry 主流程按钮。
- resume 必须绑定显式 run id，不按路径暗中复用旧 run。

### 审核

- 主画布以左右图片对比为中心，默认占据最大空间。
- 顶部显示图集、候选序号、匹配类型与置信信息。
- 元数据进入对比图下方或右侧窄面板，不挤压图片。
- 决策区固定在视口底部或图片下方：保留左图、保留右图、两张都保留、跳过图集。
- 快捷键必须与按钮文案同屏提示，并避开系统常用组合。
- overlay、缩放、拖动属于次级工具；切换后有清晰选中状态。
- 提交决策后只前进到下一候选；最后一个候选完成后展示真实汇总并提供生成计划入口。

### 导入计划

- 顶部显示计划状态、图集数、保留/排除图片数和预计大小。
- 图集作为可展开列表，展示封面和保留/总数；图片级调整位于展开内容。
- “人工复核入库计划”中可随时切换导入 / 跳过，此时仅保存 draft，不计算 plan hash。
- “锁定导入计划”是独立动作；只有锁定时才计算 plan hash 并转为 frozen。
- “确认并开始入库”只读锁定计划，不再提供导入 / 跳过调整。
- frozen 后明确只读边界；用户可以在任何文件事务创建前撤销整次尚未入库的导入任务。
- 撤销必须二次确认；成功后 import run 进入 `abandoned`、计划保留为 `invalidated` 审计记录，审核决定与源快照不删除。
- `abandoned` / `invalidated` 是 M3.8 唯一授权的状态语义扩展；不得借此改变审核、Commit、文件事务发布或 Recovery 的其余状态转换。
- 撤销成功后清空当前 workflow run 上下文并返回工作台；该任务不再提供继续审核、重新生成计划或继续提交入口，工作台下一步恢复为“开始导入”。
- 一旦存在任何文件事务证据，撤销必须 fail closed，用户继续走 Commit / Recovery 既有路径。
- 显示 plan hash 时放入诊断详情，不作为主信息。

### 入库

- 执行前用清单说明目标图库、图集数、图片数、源归档策略和不可覆盖边界。
- 主按钮文案使用“确认并开始入库”，避免模糊的“提交”。
- 执行中显示当前阶段：staging、校验、发布、数据库提交、源归档。
- 取消按钮的可用性和含义必须与真实事务阶段一致。
- 完成页展示真实结果；部分失败、recovery required 不使用成功庆祝状态。

### 恢复

- 将“可自动恢复”“需要人工处理”“历史终态”分区展示。
- 每个事务展示 run、图集、当前阶段、最后错误和后端允许的操作。
- 可恢复项才显示“继续恢复”；终态 failed/cancelled 显示检查与处置入口。
- 证据冲突或不完整时明确 fail closed，并禁止带有自动修复暗示的主按钮。

### 设置与首次配置

- 首次配置保持单列向导：选择数据库模式 → 预检 → 初始化/连接 → 完成。
- 托管本地 PostgreSQL 为默认推荐；外部 PostgreSQL 是高级选项。
- 设置页分为“数据库”“图库位置”“外观与行为”“诊断”。没有实现的设置不做占位。
- 技术探针迁入“诊断”，保留原能力和错误详情。

## 5. 响应式与窗口规则

M3 以桌面为主，不按手机页面设计，但必须支持窗口缩放：

| 宽度         | 行为                                                       |
| ------------ | ---------------------------------------------------------- |
| `>= 1280px`  | 完整侧栏；审核可显示双图 + 详情                            |
| `960–1279px` | 完整或紧凑侧栏；次级面板可折叠                             |
| `720–959px`  | 图标侧栏；统计区换行；表格折叠次级列                       |
| `< 720px`    | 最低可用兜底；审核双图改上下排列，不承诺移动端完整生产体验 |

Windows 100%、125%、150% 可作为桌面兼容性抽样矩阵；主 CTA、底部操作区和弹层不得被裁切。100% / 150% 的人工切换不属于本轮 M3.8 审查修复完成门禁，未执行时不得写成阻塞项或伪造签字。

## 6. 可访问性

- 目标 WCAG 2.2 AA。
- 全部交互可使用键盘完成，焦点顺序符合视觉顺序。
- `focus-visible` 使用清晰的 2px focus ring，不被 `overflow: hidden` 裁切。
- 图标按钮具备 accessible name；图片 alt 说明其决策角色，不重复文件名噪声。
- 对话框使用正确 focus trap、Escape 和焦点归还；优先使用 portal 或原生 dialog/popover。
- 状态更新通过合适的 live region 通知，但高频进度不得持续打断屏幕阅读器。
- 不用颜色作为唯一信息，错误与成功均有文字。

## 7. 图标与图像

- 使用同一套线性、圆角图标，建议 1.75–2px stroke；禁止混用 emoji、填充图标和多套风格。
- M3 基础状态图标由 ImageDB 在 `components/ui/StatusIcon.tsx` 中以自有 inline SVG 实现，采用 2px 圆角描边，不引入额外图标包，也不复制参考项目的图标资产。后续导航与领域图标继续沿用同一实现和打包策略。
- 图集封面与真实图片是主要视觉资产；没有图片时使用中性语义占位，不生成假照片。
- 可以为成功完成设计一个轻量的自有几何图形，但不使用 Nintendo 或参考库素材。
- 背景纹理仅可在空状态或小面积品牌区域低对比使用，不覆盖高密度工作区。

## 8. 文案规则

- 使用用户语言：“新建导入”“继续分析”“待审核”“生成导入计划”“确认并开始入库”“处理恢复”。
- 技术名词仅在需要时保留：PostgreSQL、pgvector、frozen plan、plan hash、transaction。
- 危险操作必须写明对象和后果，例如“放弃这次导入任务”；禁止只写“确定”。
- 进度和结果避免承诺未发生的动作，例如 staging 完成不能写“已入库”。
