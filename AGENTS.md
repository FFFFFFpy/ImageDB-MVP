# ImageDB 开发规则

## 项目目标

实现一个桌面图集整理应用，完成以下闭环：

```text
选择源目录
→ 扫描图集
→ 图集内部去重去相似
→ 与历史图库比较
→ 用户审核不确定候选
→ 安全写入目标图库
→ 校验文件
→ 数据库确认入库
```

## 固定技术栈

- GUI：React + TypeScript + Vite
- 桌面容器：Tauri 2
- 核心后端：Rust
- 数据库：PostgreSQL + pgvector
- 默认数据库模式：应用管理私有本地 PostgreSQL
- 高级数据库模式：连接外部 PostgreSQL

## 工作方式

1. 先阅读 `CURRENT_TASK.md`。
2. 只实现当前任务明确要求的内容。
3. 当前任务通过验收后，再进入下一任务。
4. 不提前添加未在当前任务中定义的模块、依赖、页面或抽象。
5. Commands 只负责 IPC 边界，不承载业务逻辑。
6. React 不直接访问数据库或文件系统。
7. 业务规则放在 Rust domain/services 中。
8. 数据访问统一经过 repositories。
9. 长任务必须支持进度上报和取消。
10. 状态转换集中定义，不在各模块中随意写状态字符串。

## 数据安全

1. 分析阶段不修改源文件。
2. 正式写入采用 staging、校验、发布、数据库提交的顺序。
3. 发布成功并通过完整性校验前，不归档源图集。
4. 不覆盖未知目标目录。
5. 无法可靠判断的图片交给用户审核。
6. 所有提交操作必须可恢复、可重试、保持幂等。

## 质量要求

每个任务完成时必须提供：

- 实现内容
- 修改文件
- 执行命令
- 测试结果
- 实际运行结果
- 已知限制

单元测试不能替代真实数据库和真实文件系统集成测试。
